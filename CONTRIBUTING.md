# Contributing

Thanks for contributing.

## Development flow

1. Create a branch from `main`.
2. Make your changes.
3. Run local update script:

```powershell
./atualizar_eidolons.ps1
```

4. Verify `index.html` renders correctly.
5. Open a Pull Request with a clear summary.

## Scope guidelines

- Keep changes focused and minimal.
- Avoid unrelated formatting-only edits.
- If changing parser rules, include examples in PR description.

## Commit style

Use short, clear commit messages, for example:

- `fix: improve category parsing`
- `feat: add update notification text`
- `docs: update setup instructions`
